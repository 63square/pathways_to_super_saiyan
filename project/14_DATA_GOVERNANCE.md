# 14 — Data Governance & Management Plan (DMP)

**Owner:** CSO + Head of Biostatistics (joint)
**Effective:** Upon Head of Biostat hiring (Month 6)
**Review cadence:** Quarterly

## Purpose

The program's terminal asset value lives in its longitudinal dataset. This document defines the architecture, classification, lifecycle, schema, and access controls for all data generated by Project PROMETHEUS — with special protections for the Principal's identifiable data.

This is the largest single gap in v1.0 documentation per SAB review.

---

## 14.1 Data Classification Taxonomy

Per doc 07 §7.6.1, four tiers enforced by physical and logical separation:

| Tier | Contents | Storage | Access |
|---|---|---|---|
| **Tier-Red (CORE classification)** | Principal genomic, methylation, identifiable multi-omic, biopsy results, full medical record | TEMPEST-rated SCIF-equivalent enclave at Próspera HQ; WORM optical media only; air-gapped (no network interface); threshold cryptography 3-of-5 hardware-key shares in separated jurisdictions | Biometric + hardware key + 2 of 3 authorized parties co-present; all access logged immutably |
| **Tier-Amber (SECRET classification)** | Routine labs, wearables, non-identifiable derivatives, lab-research data | Segmented clinical VLAN; one-way data diode push to Tier-Red; encrypted at rest | Need-to-know clinical/research staff |
| **Tier-Green (CONFIDENTIAL classification)** | Spinout pre-clinical, anonymized research datasets, public-pre-registration data | Standard enterprise data lake | Program staff with NDA |
| **Tier-Open (RESTRICTED classification)** | Pre-registered protocols (post-M0 public filing), peer-reviewed publications, defensive publications | Public registries (OSF, ClinicalTrials.gov-equivalent) | Public |

Four-zone segmentation enforced by physical firewalls with documented ACLs. **Quarterly red-team explicitly testing lateral movement between zones.**

## 14.2 Data Lifecycle

### Collection
- All Tier-Red and Tier-Amber data is captured against a **pre-registered Case Report Form (CRF)** with locked schema
- CRF design follows CDISC SDTM standard for transferability
- Source documents (lab reports, imaging) ingested via HL7/FHIR through inspected proxy
- Wearable data ingested via vendor APIs with cryptographic signing

### Processing
- All transformations (normalization, derivation, harmonization) are **scripted and version-controlled** — no manual edits to raw data
- Audit trail captures who, when, why for every transformation
- Tier-Amber → Tier-Red ingestion via one-way data diode only

### Retention
- **Minimum 50 years** for all Principal data, decision memos, EEP reports, gate determinations, recusal logs
- Three jurisdictionally separated copies: Cayman (primary), Cook Islands (backup), US/EU academic deposit (escrowed institutional repository, sealed)
- Annual integrity audit (cryptographic hash verification)

### Destruction
- **Principal right-to-be-forgotten:** can demand destruction at any time, with GC opinion on legal/insurance retention requirements
- NSA/CSS 9-12 wipe standard for digital
- Cross-cut + incineration for paper at SECRET+
- Destruction events documented; CSO + GC + Head of QA co-sign
- Biological samples per advance directive

## 14.3 Schema & Ontology Governance

- **Master Data Model** managed by Head of Biostatistics
- 47 baseline biomarkers + 11 KPIs (locked at M0 public pre-registration)
- ~3000+ proteins (Olink Explore), full metabolomic + lipidomic, microbiome shotgun, methylome 450K, WGS
- Standard ontologies: SNOMED-CT (clinical), LOINC (labs), HPO (phenotype), RxNorm (drugs), Gene Ontology, ChEBI
- Schema changes require change-control workflow (doc 17) + Head of Biostat sign-off

Linkage table mapping pseudonymized IDs to Principal identity is **only** in Tier-Red enclave. Cannot be reconstructed from any subset of Tier-Amber or Tier-Green.

## 14.5 FAIR Principles for Shareable Data

For spinout-shareable data (Tier-Green) and public-pre-registration data (Tier-Open):

- **Findable**: DOI assigned; metadata indexed
- **Accessible**: Defined access protocol per tier
- **Interoperable**: Standard ontologies + open formats (Parquet, CRAM, FHIR)
- **Reusable**: License terms documented per dataset

Principal-identifiable data is NOT subject to FAIR — restricted access is the security architecture.

## 14.6 Chain-of-Custody for Biospecimens

Every biospecimen (blood, urine, stool, biopsy, sperm cryo, mitochondrial isolate):

- **Barcoded at collection** with Tier-Red pseudonymized ID
- **Custody log** electronically signed at every handoff (phlebotomist → lab → freezer → analysis → disposal)
- **Cryostorage temperature monitoring** with alarm escalation
- **Annual inventory audit** by Head of QA
- **Disposal under advance directive instructions** with two-witness verification

Biospecimens linkable to Principal identity stored only at Tier-Red Próspera facility (with backup at UAE Tier-Red equivalent).

## 14.7 eCRF / EDC Validation

- **21 CFR Part 11–equivalent** electronic records compliance
- EDC system validated per GAMP 5
- Audit trail: who, when, what changed, why
- Electronic signatures with multi-factor authentication
- Periodic system validation reviews

## 14.8 Data-Sharing Agreements

For external parties (CROs, spinouts, academic collaborators):

| Recipient | DSA Requirements |
|---|---|
| Phase 0 CRO (vendor running Olink, Metabolon, etc.) | Vendor V2 access (de-identified Principal data, SOC 2 Type II, named-individual NDA, annual on-site audit) |
| GMP CDMO (AAV mfg Phase 1) | Vendor V3 (TS-equivalent vetting of individuals; FOCI review of entity; contractual right of CI investigation; no subcontracting without consent) |
| Spinout (each) | **EEC binding pre-approval per data-transfer instance**; field-limited; royalty-bearing; reversion on spinout failure |
| Academic collaborator | Limited de-identified Tier-Green; publication review SOP (60-day IP review pre-submission); liquidated damages for premature disclosure |
| Public registry (OSF) | Pre-registered protocols + KPI thresholds only (integration thesis redacted) |

## 14.9 Cross-Border Data Flow Architecture

Per doc 07 §7.6.2:

- **Single legal-basis architecture:** Principal as data subject signs explicit GDPR Art. 9(2)(a) consent naming every recipient jurisdiction
- **Intragroup DTA** with 2021 SCCs + Transfer Impact Assessment (post-Schrems II)
- **HIPAA BAAs** for Boston-side vendors; Boston entity kept in research-only posture under Common Rule (not HIPAA-covered)
- **Pseudonymization-at-ingest** for Boston subset — never holds re-identifiable Principal data
- **Cayman DPA 2017** transfer adequacy review
- **UAE Federal Decree-Law No. 45/2021** compliance
- **Honduran Ley de Protección de Datos Personales** compliance

## 14.10 Statistical Analysis Plan (SAP) Lock

- **SAP authored by Head of Biostatistics** (independent of CSO bias)
- SAP **locked at M0** before any data is collected, filed with EEP
- SAP changes require: EEP approval + SAB non-objection + LID sign-off
- All analyses run against locked SAP
- Negative results published internally; considered in pivot decisions
- No post-hoc threshold changes

## 14.11 Data Quality Control

Operational KPIs (doc 08 §8.5 expanded):
- Query rate per CRF (<2 per 100 fields)
- Data lock latency (<14 days post-collection)
- Source data verification rate (>10% sample)
- Pseudonymization integrity (zero re-identification incidents)
- Tier-Red access audit (quarterly review by Head of QA)

PMO audits randomly selected records monthly. Head of Biostat reviews data integrity quarterly. External auditor reviews annually.

## 14.12 Backup, Recovery, Continuity

- **Tier-Red:** 3 geographic copies (Próspera + UAE + Cayman trustee escrow). Annual restore test.
- **Tier-Amber:** Cloud + on-prem hybrid; 4-hour RPO, 24-hour RTO.
- **Tier-Green/Open:** Standard enterprise backup.

In wind-down scenario (doc 17 BCP): asset preservation sequence pre-defined. Principal data destruction per advance directive.

## 14.14 Incident Response

Data incidents categorized:
- **Cat-1 Tier-Red breach (real or suspected):** Crisis protocol C8 (doc 07); CSecO + GC + CMO + LID notified within 1 hour; FBI/Interpol notification if external attribution suspected
- **Cat-2 Tier-Amber breach:** CSecO + GC + CMO within 4 hours; notification per GDPR / state breach laws
- **Cat-3 Tier-Green breach:** CSecO + GC within 24 hours
- **Cat-4 Data integrity event (corruption, loss):** Head of QA + Head of Biostat within 4 hours

All incidents logged. Lessons fed into CAPA. Quarterly incident report to LID + Risk Committee.
